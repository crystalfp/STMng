Notes

  1. The real format derives from the source code of the VMD plugin
  2. I remember a difference between what has been described and the real file format
  3. The plugin read only coordinates from a $DATA section with symmetry C1
  4. An old version of PDB, prior to 1992. The ‘Temperature factor’ field starts at column 71 instead of 61. Should be documented somewhere…
  5. Seems no more active
  6. In the unit cell file each block of three lines contains three floats and refers to one frame. The nine numbers define the three base vectors (one per line).
  7. Read the old and new versions (the one with element names). Write only the new version.
  8. The energy file is a table with two lines of column headers then one line for each structure in the associated POSCAR file. The column containing the enthalpies has one energy or enthalpy value per line. The energy value is per atom or per structure as defined in the second header line.
  9. The energy file has one energy or enthalpy value per line and there are as many lines as the number of structures in the associated POSCAR file. The energy value is per atom or per structure. The write is done from the fingerprint scatterplot window for the selected structures and the energy is per atom.
  10. The only documentation of this format is in this VMD plugin. This format seems produced by VASP 5.2 for which no CONTCAR (or POSCAR) file is needed. In the fixed cell variant it is a POSCAR file followed by a separation line and a set of atoms coordinates for the remaining steps. Instead, the variable cell variant is a concatenation of POSCAR formatted steps.
