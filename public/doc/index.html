See The Molecule new generation (STMng) is a chemical and crystallographic structures visualization and analysis tool that implements some of the functionalities of the old STM4 visualization tool.

STMng primary use is to analyze the results of the USPEX computational material discovery system.

Unlike its predecessor, STMng is built on top of open source framework and libraries.

STMng screen

The main STMng screen is divided in two areas: on the right the Viewer and on the left the Node controls panel.

Clicking on the separator between the two areas (the Gutter) hide the node control panel. Clicking again on it, that now is on the left of the full screen viewer, unhide the panel.

On the title bar of the window is visible the currently loaded project: the project file name or the “default project” mark. If a file is loaded in the Structure reader, the filename is also shown.

The Node controls panel contains the Node selector that selects the node for which the controls are shown. Below it are located the parameter’s controls specific for the selected node. At the bottom there are the controls to Manage camera position that resets the camera position in the viewer and orients the camera along various axis.

These controls show by default the “Reset camera” button that resets the camera position to show the loaded structure. On its left side the button show the next set of buttons that align the camera with the XYZ axis. Here the button reverse the chosen axis. If the structure in the viewer has an unit cell defined, on the left side the button show the next set of buttons that align the camera with the ABC cell axis. In this set of buttons and in the previous one if the cell is not present, the button returns to the first controls.

Some nodes and menu entries open a secondary window that can be positioned freely and resized with data and controls specific for its functionality.

Project: the node graph description

The project is the description of the nodes used, their interconnections and their parameters.

The nodes provided by the project have their UI visible in the UI section of the screen.

The application comes with an hardcoded project, called the default project, that contains all the nodes shipped. It cannot be changed, but can be edited and saved as a different project.

The current project or the default project are automatically loaded when the application starts. The project file could also be loaded from the command line.

A project is a JSON formatted file with extension .stm containing a JSON record with the following keys:

{ "graph": { // List the loaded nodes and their interconnections }, "viewer": { // Optional section to collect the viewer 3D parameters }, "ui": { // Optional section that collects the parameters for each node } }

Graph section

This section is a JSON record with the unique identifier of the nodes as keys and, for example, the following content:

{ "graph": { "draw": { // The unique identifier of the node "label": "Draw structure", // The label that appears in the selection panel "type": "draw-structure", // The type of node, that is, its functionality "x": 510, // The (optional) X position of the node in the project editor "y": 210, // The (optional) Y position of the node in the project editor "in": "reader" // The id of the node from which this node takes the input. // It is missing if the node has no input or if the input // comes only from the graphical scene }, // Other nodes }, . . . }

Project editor

The graph structure could be seen using the File → Project editor menu entry. This menu entry opens the Project editor. This editor allows the modification of the currently loaded project that, at the end, could be saved and becomes the current project. Also an entirely new project could be created and saved.

Interaction with the viewers

Interacting with the viewer means moving the camera around the visualized object.

This interaction is done through mouse movements and using the keyboard arrows. Note that the same mouse actions are applicable to the Energy Landscape and Compare Structures viewers, but not the keyboard arrows shortcuts.

Terminology

Rotate Rotate the camera around the scene center
Pan Move the camera on the viewer plane
Zoom Change the magnification of the camera without moving it
Dolly Move the camera perpendicular to the viewer plane

Mouse

Drag with left mouse button Rotate
Drag with right mouse button Pan
Drag with center mouse button Dolly
Mouse wheel Zoom

Keyboard arrows

Any arrow Pan
Control + any arrow Rotate
Shift + any arrow Zoom (Left and Down do the same, as do Right and Up)
Alt + any arrow Dolly (Left and Down do the same, as do Right and Up)

Dolly movement have a visible effect only with Perspective camera. Instead with Orthographic camera objects too near or too far from the camera are cut from view.

Command line options Usage: STMng [options] [project-file] See The Molecule new generation is a chemical and crystallographic structure visualizer that inherits some functionalities of STM4 Arguments: project-file project file to be loaded Options: -V, --version output the version number -t, --theme <theme> user interface theme (choices: "dark", "light") -d, --default force load of default project -v, --verbose verbose -e, --enable enable developer tools in production build -x, --extra <switches> extra command line switches -h, --help display help for command

The “-x” command line switch pass the subsequent switches to the internal processes. If more than one, they are comma separated and if they pass a value, switch and value are separated by a “=” sign.

Verbose can be set also with the “STM_NG_VERBOSE” environment variable set to any value.

Developer tools can be enabled in production also with the “STM_NG_ENABLE” environment variable set to any value.

Technology

STMng is an ElectronJS multiplatform application written in Typescript with small interface parts written in C++.

The user interface is based on the Vue framework with Vuetify widgets.

The 3D viewer uses the three.js interface to WebGL provided by the Chromium browser integrated inside ElectronJS.

The development toolchain is based on Vite, the final build for production uses electron-builder. The bundling of the code for the parallel computations is done by Rollup.

Credits for specific libraries used are listed in the respective node documentation pages.
