Fingerprints

Compute fingerprints for a set of structures to group and analyze them

The module implements the fingerprinting method to group similar structures as presented in:

  • Valle, M., & Oganov, A. R. (2008). Crystal Structures Classifier for an Evolutionary Algorithm Structure Predictor. IEEE Symposium On Visual Analytics Science and Technology, VAST 2008, 11–18, DOI: 10.1109/VAST.2008.4677351
  • Oganov, A. R., & Valle, M. (2009). How to quantify energy landscapes of solids. J. Chem. Phys. 14 March 2009; 130 (10): 104504, DOI: 10.1063/1.3079326
  • Valle, M., & Oganov, A. R. (2010). Crystal fingerprint space – a novel paradigm for studying crystal-structure sets. Acta Crystallographica Section A, 66(5), 507–517, DOI: 10.1107/S0108767310026395

Controls

The controls are grouped in logical steps that normally should be completed top to bottom.

Accumulated structures

Structures loaded readonly text Count of accumulated structures. The start and stop of the accumulation are done from the Structure Reader page.
Load one button Load the current structure only. Useful to compute the fingerprint of a single structure that is not part of a sequence of structures.
Reset button Clear the accumulated structures.

Filter structures

Filter by energy per atom toggle From the accumulated structures extract only the ones that satisfy the following constrains on the energy per atom.
Threshold from minimum energy toggle The maximum energy threshold is computed from the minimum per atom energy of accumulated structures.
Energy from minimum / Max energy numeric field For filtering structures enter the maximum energy per atom the structures should have or, when “Threshold from minimum energy” is set, the offset from the minimum energy per atom of the accumulated structures to establish the threshold.
Max energy readonly numeric field The computed or manually entered energy threshold.
Structures selected readonly text Count of structures eventually selected by energy over the total number of structures.

Compute fingerprints

Structures are nanoclusters toggle Mark the structures as nanoclusters, that is, they are not crystals with a periodic structure.
Force cutoff at: toggle + numeric field When set, forces the cutoff distance for the computation of fingerprint to the value entered.
Cutoff readonly text The cutoff distance (computed or manually entered) that will be used by the fingerprint computation.
Fingerprinting method list Select the method for computing the fingerprints:
Oganov-Valle fingerprint
The fingerprint is composed by a section for each atom type in the unit cell. Each section is an ascending ordered set of distances from the corresponding atom to all the other atoms of the extended unit cell. All these sets contain the same number of distances.
Oganov-Valle per site fingerprint
It is the same as “Oganov-Valle fingerprint” but the centroid of all fingerprints is removed from each fingerprint.
Dot-matrix fingerprint
This method takes three atoms, one (A) in the unit cell and two (B & C) in the extended unit cell (cells added around the unit cell to cover the cutoff distance). Then fill a square matrix in which row and column indices are computed from the AB and AC distances. The accumulated value is:

The matrix is then transformed to a vector appending the diagonals of the upper triangle starting from the longest one. Note: this method is quite slow.
Peak width numeric field The distance values are smoothed with a gaussian of this full width at half maximum (FWHM) value and then binned.
Bin size numeric field To convert smoothed distance peaks into a vector of values, that is, the fingerprint, the smoothed interatomic distances are binned to this size.
Multi process parallelism toggle Select to parallelize computation using multiple processes. If not set the parallelization is done using threads in a single process. The multi-process method is slower, but gives more memory to each computation. Select it if selecting parallelization on threads has memory problems.
Compute fingerprints & distances button Compute the fingerprints for the selected structures and their distances.
Fingerprint dimension readonly text At the end of the computation shows the length of the computed fingerprints, that is, the dimensionality of the space containing the fingerprints.
Intrinsic dimension & theoretical value readonly text Estimation of the dimensionality of the embedded fingerprint space (the subspace containing the fingerprints that is embedded in the fingerprint space). The estimation is done using a Maximum Likelihood Estimator that provides local dimension estimates. The theory gives a value of 3 * numAtoms + 3 (three coordinates per atom, less three for an atom fixed at the origin, plus six cell parameters).
Local dimension range readonly text Range of the local intrinsic space dimensions around each fingerprint.

Compute distances

Distance method list Select the measuring method to obtain the distance between fingerprints:
Cosine distance
Cosine distance between fingerprints interpreted as multidimensional vectors. The distances are scaled between 0 and 1. This measure violates the triangle inequality.
Euclidean distance
Euclidean distance between fingerprints interpreted as points in a multidimensional space.
Minkowski distance of order ⅓
The Minkowski distance with order p = ⅓. With this order value, the measure violates the triangle inequality but spread points better in the multidimensional space facilitating grouping.
Fix triangle inequality toggle Iteratively adjust distances to satisfy the triangle inequality for every triples of distances.
Distances computed readonly text At the end of the computation shows the number of computed distances.
Fix inequality readonly text At the end of the computation with "Fix triangle inequality" set shows the result of the fixing.

Remove duplicates

Remove toggle Enable removal of duplicated structures, that is, structures distant less than the distance threshold.
Distance threshold numeric field The maximum distance between two fingerprints to consider the highest energy one for removal.
Points removed readonly text At the end of the computation show the number of structures removed over the total.

Group similar

Grouping method list Select the method to group similar fingerprints:
Pseudo SNN
Structure clustering method based on the number of nearest neighbors points shared between pairs of connected points and confirm this last connection only if the number is at least K (equal to one plus the margin value). SNN means Shared Nearest Neighbor and is “Pseudo” becouse laks the final DBSCAN step to refine clustering.
Hierarchical grouping (single linkage)
It is a methods of hierarchical clustering based on grouping clusters in bottom-up fashion, at each step combining two clusters that contain the closest pair of elements not yet belonging to the same cluster as each other.
Hierarchical grouping (complete linkage)
It is an agglomerative hierarchical clustering that at the beginning of the process, has each element is in a cluster of its own. The clusters are then sequentially combined into larger clusters until all elements end up being in the same cluster.
Distance thresh. numeric field Distance threshold for two structures to consider them in the same group. The distance is given as a percentage of the global maximum distance.
Margin numeric field For the Pseudo SNN method it is the number of connection above one to consider two structures connected.
Group similar structures button Classify fingerprints into groups of similar ones.
Groups found readonly text At the end of the computation shows the number of groups found.
